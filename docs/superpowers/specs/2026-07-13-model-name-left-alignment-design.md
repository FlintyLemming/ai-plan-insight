# 模型用量表模型名左对齐设计

## 目标

模型用量表中的模型名单元格在手机端和桌面端统一靠左对齐。模型名表头继续靠左，token 明细和总计等数值列继续靠右。

## 现状

`ai_plan_insight/index.html` 中的通用表格规则将所有 `th` 和 `td` 设为右对齐。现有样式只覆盖了 `th.col-model`，因此模型名表头靠左，而 `td.col-model` 仍继承右对齐，导致表头与内容错位。

## 方案

为模型名数据单元格增加局部样式：

```css
.usage-table td.col-model {
  text-align: left;
}
```

该规则仅改变模型名列的数据单元格，不修改其他列、表格结构、分页逻辑或响应式断点。内部色块和模型名仍使用现有的 inline-flex 布局。

## 测试与验收

1. 先在 `tests/test_index_usage_view.py` 添加回归测试，要求模型名数据单元格存在明确的左对齐规则，并确认测试在修改 CSS 前失败。
2. 添加最小 CSS 修改后，运行该测试文件并确认通过。
3. 运行相关测试集，确认模型用量页其他行为没有回归。
4. 在手机端和桌面端检查计算后的样式：模型名单元格为 `text-align: left`，数值单元格仍为 `text-align: right`。

## 非目标

- 不改变表格列宽、字体、颜色或间距。
- 不改变模型排序、分页或截断逻辑。
- 不调整图表、图例或其他页面。
